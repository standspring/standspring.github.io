Git은 개발자가 가장 많이 사용하는 **버전 관리 시스템(Version Control System)** 입니다.

쉽게 말하면:

* 파일 변경 이력을 저장하고
* 이전 상태로 되돌릴 수 있고
* 여러 사람이 동시에 작업할 수 있게 해줍니다.

즉, **"코드의 타임머신"** 이라고 생각하면 됩니다.

---

## 왜 Git을 써야 할까?

예를 들어 이런 상황이 자주 생깁니다.

```plaintext
project_final.py
project_final_real.py
project_final_real_last.py
project_final_real_last_fix.py
```

파일 이름이 점점 이상해집니다.

Git을 쓰면:

```plaintext
언제
누가
무엇을
왜 바꿨는지
```

모두 기록됩니다.

---

## Git과 GitHub 차이

많이 헷갈리는 부분입니다.

### Git

내 컴퓨터 안에서 버전 관리하는 프로그램

### GitHub

Git 저장소를 인터넷에 올려두는 서비스

즉:

```plaintext
Git = 엔진
GitHub = 클라우드 저장소
```

---

## Git 설치

공식 사이트:

[https://git-scm.com](https://git-scm.com)

설치 후 확인:

```bash
git --version
```

정상 설치되면 버전이 표시됩니다.

예:

```bash
git version 2.48.1
```

---

## 최초 1회 사용자 설정

Git은 누가 작업했는지 기록해야 하므로 이름과 이메일을 설정합니다.

```bash
git config --global user.name "홍길동"
git config --global user.email "your@email.com"
```

확인:

```bash
git config --list
```

---

## Git 저장소 시작하기

작업 폴더 이동:

```bash
cd my_project
```

Git 시작:

```bash
git init
```

그러면 `.git` 폴더가 생성됩니다.

이 순간부터 Git이 변경사항을 추적합니다.

---

## 파일 상태 확인

현재 상태 보기:

```bash
git status
```

예:

```plaintext
untracked files:
main.py
```

뜻:

Git이 아직 추적하지 않는 파일

---

## 파일 추가 (Stage)

Git은 바로 저장하지 않고 먼저 올려둡니다.

```bash
git add main.py
```

전체 추가:

```bash
git add .
```

---

## 저장하기 (Commit)

실제 기록 저장:

```bash
git commit -m "main.py 최초 생성"
```

설명:

```plaintext
-m = 메시지 작성
```

좋은 메시지 예시:

```plaintext
로그인 기능 추가
버그 수정
README 작성
```

---

## 변경 이력 보기

```bash
git log
```

간단히 보기:

```bash
git log --oneline
```

예:

```plaintext
a1b2c3d 로그인 기능 추가
d4e5f6g README 작성
```

---

## 수정 후 다시 저장 흐름

항상 이 순서입니다:

```plaintext
수정
→ add
→ commit
```

예:

```bash
git add .
git commit -m "버튼 클릭 버그 수정"
```

---

## 이전 상태와 비교

```bash
git diff
```

어떤 줄이 바뀌었는지 보여줍니다.

---

## 실수했을 때 되돌리기

### add 취소

```bash
git reset HEAD 파일명
```

### 수정 취소

```bash
git checkout -- 파일명
```

주의:

수정 내용이 사라집니다.

---

## 브랜치(branch) 개념

브랜치는 독립 작업 공간입니다.

현재 작업 중 메인 코드에 영향 없이 실험 가능

생성:

```bash
git branch feature1
```

이동:

```bash
git checkout feature1
```

생성과 동시에 이동:

```bash
git checkout -b feature1
```

---

## 브랜치 확인

```bash
git branch
```

현재 위치:

```plaintext
* feature1
  main
```

---

## 브랜치 합치기 (merge)

main으로 이동:

```bash
git checkout main
```

합치기:

```bash
git merge feature1
```

---

## GitHub 연결하기

원격 저장소 연결:

```bash
git remote add origin https://github.com/아이디/저장소.git
```

확인:

```bash
git remote -v
```

---

## GitHub 업로드(push)

최초:

```bash
git push -u origin main
```

이후:

```bash
git push
```

---

## GitHub에서 내려받기(clone)

```bash
git clone 저장소주소
```

예:

```bash
git clone https://github.com/아이디/project.git
```

---

## 최신 내용 가져오기(pull)

```bash
git pull
```

---

## 가장 많이 쓰는 핵심 5개만 기억하기

```bash
git status
git add .
git commit -m "메시지"
git pull
git push
```

이 5개면 대부분 작업 가능합니다.

---

## 초보자가 가장 많이 하는 실수

### 1. commit 전에 add 안함

```bash
git commit -m "저장"
```

안 됨

반드시:

```bash
git add .
git commit -m "저장"
```

---

### 2. 너무 큰 commit

좋지 않음:

```plaintext
모든 수정 완료
```

좋음:

```plaintext
로그인 버튼 추가
로그인 버그 수정
```

---

### 3. main에서 바로 작업

추천:

기능별 branch 사용

---

## 추천 Git 작업 습관

```plaintext
작게 수정
자주 commit
메시지는 명확하게
```

## VS Code에서 Git 쓰면 더 쉬움

VS Code 좌측 Source Control 탭 사용 가능

가능한 작업:

* 변경 파일 보기
* add
* commit
* push
* branch 변경

터미널보다 초보자에게 훨씬 쉽습니다.

---

## 정리

Git은 처음엔 어렵지만 실제 사용하는 명령은 많지 않습니다.

핵심 흐름:

```plaintext
수정
→ add
→ commit
→ push
```

이 흐름만 익히면 금방 익숙해집니다.
